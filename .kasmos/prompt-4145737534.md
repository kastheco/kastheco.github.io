Keep the content contract and sample data together so one coder pass establishes the post id convention, draft filtering rules, tag normalization, and fixture content used by later route/build verification.

1. Create `blog/src/content.config.ts` using Astro's current content-layer entrypoint, not the planner's obsolete nested path. Use `defineCollection` from `astro:content`, `glob` from `astro/loaders`, and `z` from `astro/zod`:
```ts
import { defineCollection } from 'astro:content';
import { glob } from 'astro/loaders';
import { z } from 'astro/zod';

const posts = defineCollection({
  loader: glob({ base: './src/content/posts', pattern: '*.md' }),
  schema: z.object({
    title: z.string().min(1),
    description: z.string().min(1),
    pubDate: z.coerce.date(),
    updated: z.coerce.date().optional(),
    tags: z.array(z.string()).default([]),
    draft: z.boolean().default(false),
  }),
});

export const collections = { posts };
```
Use `pattern: '*.md'`, not `**/*.md`, to enforce the flat-file v1 contract at the loader boundary.
2. In `blog/src/lib/posts.ts`, centralize every collection query and post URL helper. Export this surface:
```ts
import type { CollectionEntry } from 'astro:content';

export type PostEntry = CollectionEntry<'posts'>;

export async function getPublishedPosts(
  options?: { includeDrafts?: boolean }
): Promise<PostEntry[]>;

export function getPostPath(postId: string): string;

export function formatPostDate(date: Date): string;
```
`getPublishedPosts()` must be the only place that calls `getCollection('posts')`. Default `includeDrafts` to `!import.meta.env.PROD`, filter drafts there, and sort newest-first by `pubDate`. `getPostPath()` must always return `/blog/posts/<id>/` so routes, tags, and RSS never hard-code that URL shape separately.
3. In `blog/src/lib/tags.ts`, export a deterministic tag helper layer:
```ts
import type { PostEntry } from './posts';

export type TagSummary = {
  label: string;
  slug: string;
  count: number;
};

export function slugifyTag(tag: string): string;

export function getTagSummaries(posts: PostEntry[]): TagSummary[];

export function getPostsForTag(posts: PostEntry[], tagSlug: string): PostEntry[];
```
Normalize tags with lowercase + hyphenation, collapse repeated whitespace and punctuation, count each normalized tag at most once per post, and if two labels map to the same slug keep the first label encountered while merging counts. Return summaries in a deterministic order.
4. Add `blog/src/content/posts/welcome-to-the-blog.md` with schema-valid frontmatter, `draft: false`, and at least one `[[obsidian-syntax-demo]]` wikilink so the happy-path post resolver is exercised.

6. Add a small deterministic SVG at `blog/public/images/posts/obsidian-syntax-demo/demo-terminal.svg`. Keep it text-based so it is easy to diff and does not require any extra asset pipeline.
